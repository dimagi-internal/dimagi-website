Nouveaux modèles d’outils BI pour soutenir la riposte à la COVID-19

Des modèles de tableaux de bord Excel, Power BI et Tableau pour transformer rapidement les données CommCare en informations exploitables sur la COVID-19.

D
Dimagi Jul 2020 6 min read Global Health
Share LinkedIn X

Le tableau de bord Tableau créé avec le Ministère de la Santé du Togo a inspiré le modèle BI de Dimagi pour la COVID-19.

Nous savons combien il est essentiel que les données COVID-19 soient immédiatement disponibles et exploitables. Pour que les gouvernements et les organisations puissent fournir des efforts COVID-19 efficaces et ciblés, il faut que ces efforts soient axés sur les données.

Pour aider nos partenaires à mettre rapidement en place leurs pipelines de données COVID-19, Dimagi développe des modèles de tableaux de bord pour les outils BI (informatique décisionnelle, business intelligence en anglais) les plus courants de nos partenaires, qui peuvent être facilement connectés grâce aux fonctions d’intégration de CommCare.

À la suite de la publication du modèle d’application FFX COVID-19, nous avons créé des modèles de tableaux de bord téléchargeables pour Excel, Tableau et Power BI. Ces tableaux de bord sont censés servir de points de départ de l’utilisation par les organisations de leurs données CommCare sur la COVID-19. Les données CommCare peuvent être introduites dans ces outils, soit en exploitant les fonctions d’intégration de CommCare, soit en utilisant des exportations de données produites manuellement.

Excel

Dans le cadre de la panoplie de modèles FFX de l’OMS, nous avons créé un modèle de tableau de bord Excel à télécharger et à utiliser par toute organisation déployant l’application FFX de l’OMS.

Le tableau de bord Excel présente un résumé de la situation actuelle des cas confirmés ou suspects, ainsi que les hospitalisations, les guérisons et les décès déclarés. Il affiche également les tendances dans le temps pour l’ensemble des cas et les nouveaux cas quotidiens, ainsi que des rapports sur le respect du protocole FFX pour le suivi des cas.

Ceci n’est qu’un exemple du type d’informations que vous pouvez extraire de votre modèle d’application FFX. Consultez les descriptions des indicateurs et des rapports sur cette page, et inspirez-vous en pour développer et personnaliser votre tableau de bord afin de répondre le mieux aux besoins de votre équipe.

Si vous êtes abonné à une version Standard ou à une version supérieure de CommCare, vous pouvez utiliser la fonction d’intégration du tableau de bord Excel de CommCare pour synchroniser automatiquement les données CommCare avec Excel. Si vous disposez d’un abonnement Community, vous pouvez toujours utiliser ce tableau de bord et coller vos données à partir de votre exportation de cas CommCare. Vous trouverez ici les instructions pour les étapes de configuration.

Power BI

Nous avons également développé un modèle de tableau de bord Power BI pour le modèle d’application FFX de l’OMS basé sur le même ensemble d’indicateurs de base.

Le tableau de bord Power BI présente des rapports sur les mêmes indicateurs de base que le tableau de bord Excel, y compris la situation actuelle des cas et les tendances dans le temps, mais il exploite certaines des fonctionnalités plus avancées de Power BI, comme les cartes interactives basées sur les données GPS des cas. Voir les descriptions des indicateurs et des rapports sur cette page.

Si vous êtes abonné à une version Advanced ou une version supérieure de CommCare, vous pouvez utiliser l’intégration de flux Odata de CommCare pour synchroniser automatiquement les données CommCare avec votre tableau de bord Power BI. Si vous disposez d’un abonnement Community, vous pouvez toujours utiliser ce tableau de bord et vous servir d’une exportation de données CommCare téléchargée manuellement comme source de données. Vous trouverez ici les instructions pour les étapes de configuration.

Tableau

Nous avons également publié une version Tableau du modèle de tableau de bord FFX de l’OMS, qui est accessible via notre page Tableau Public.

Le tableau de bord Tableau comporte les mêmes rapports que le tableau de bord Power BI, plus quelques rapports supplémentaires sur les taux de mortalité par groupe d’âge et les tests, qui peuvent tous être produits à partir des données du modèle d’application FFX. Voir les descriptions des indicateurs et des rapports sur cette page.

Les options d’intégration de CommCare pour Tableau sont les mêmes que pour Power BI : vous pouvez intégrer automatiquement si vous disposez d’un abonnement à une version Advanced ou supérieure, ou utiliser les exportations Excel si vous disposez d’un abonnement Community. Vous trouverez ici les instructions pour les étapes de configuration.

À mesure que nous continuerons à développer notre modèle de panoplie d’outils, nous maintiendrons les informations les plus récentes sur les modèles sur notre page Données pour COVID-19.

BI en action : tableau de bord du Ministère de la Santé du Togo

Nos modèles de tableaux de bord bénéficient d’une forte boucle de rétroaction avec nos partenaires. Leur développement est fortement influencé par notre travail avec les organisations qui déploient nos outils de riposte à la COVID-19.

Depuis le mois de mars 2020, Dimagi soutient le Gouvernement du Togo dans ses efforts de riposte à la COVID-19. Dimagi a adapté le modèle d’application de recherche des contacts FFX de l’OMS au contexte du Togo et a développé un tableau de bord Tableau présentant des rapports sur des indicateurs critiques de COVID-19.

Déploiement
1 semaine

Le temps qu’il a fallu pour mettre en place un tableau de bord affichant les indicateurs prioritaires du Ministère de la Santé du Togo : nombre total de cas, tendances dans le temps et répartition géographique des cas suspects et confirmés.

Le développement du tableau de bord du Togo a conduit à la création de notre modèle de tableau de bord Tableau décrit ci-dessus. Grâce à ce travail de base avec le Ministère de la Santé du Togo, le modèle de tableau de bord Tableau est maintenant disponible pour être rapidement adopté par d’autres programmes afin d’utiliser immédiatement leurs données COVID-19.
